❓ What is ss?

ss is a Linux command that displays socket information.

It is the modern replacement for:

netstat

Unlike netstat, ss retrieves information directly from the kernel and is generally faster, especially on systems with many connections.

Common Mistakes
❌ Thinking a Listening Port Means the Application Is Healthy

A process may be listening but still fail requests.

For complete health verification:

Check application logs
Test the endpoint
Verify readiness and liveness probes in Kubernetes
❌ Confusing LISTEN with ESTAB
LISTEN means the application is waiting for connections.
ESTAB means a TCP connection has already been established.
❌ Expecting Every Kubernetes Component to Appear in ss

Some components communicate primarily through Unix domain sockets or other mechanisms, and not every component will have a TCP listening port visible in ss.

💡 Key Takeaways

✅ ss is the modern Linux tool for inspecting network sockets.

✅ It can display listening ports, established connections, TCP states, and socket statistics.

✅ sudo ss -ltnp is one of the most useful commands because it shows which process owns each listening port.

✅ ss is invaluable for troubleshooting Spring Boot, PostgreSQL, Docker, and Kubernetes networking problems.

✅ Combining ps, top, jstack, jcmd, jmap, and ss provides visibility across processes, JVM internals, memory, and network connectivity.
