# ❓ What Is `ps`?

`ps` stands for **Process Status**.

It reads information from the Linux kernel's process table (primarily through the `/proc` filesystem) and displays running processes.

Example:

```bash
ps
```

Output:

```text
 PID TTY          TIME CMD
1234 pts/0    00:00:00 bash
2345 pts/0    00:00:00 ps
```

---

# 🧠 What Is a Process?

A process is a running program.

Examples:

```text
java
python
node
nginx
postgres
bash
```

Each running program receives:

* PID
* Virtual memory
* CPU scheduling
* Open files
* Environment variables
* File descriptors

---

# 🆔 Process ID (PID)

Every process has a unique Process ID.

Example:

```text
PID

1204

↓

java
```

Another process:

```text
PID

2058

↓

nginx
```

PIDs uniquely identify processes while they are running.

---

# 👨‍👩‍👧 Parent and Child Processes

Processes are created by other processes.

Example:

```text
systemd (PID 1)
      │
      ▼
containerd
      │
      ▼
containerd-shim
      │
      ▼
java
```

The parent process ID (PPID) shows which process created another process.


# 📊 Important Columns

Example:

```bash
ps -ef
```

| Column | Meaning                    |
| ------ | -------------------------- |
| UID    | User running the process   |
| PID    | Process ID                 |
| PPID   | Parent Process ID          |
| C      | CPU scheduling information |
| STIME  | Process start time         |
| TTY    | Terminal                   |
| TIME   | CPU time consumed          |
| CMD    | Executed command           |

---

# 💾 Memory Information

Using:

```bash
ps aux
```

Important columns:

| Column | Meaning                          |
| ------ | -------------------------------- |
| VSZ    | Virtual memory size              |
| RSS    | Resident memory currently in RAM |
| %MEM   | Percentage of system memory used |

Example:

```text
java

RSS = 850 MB
```

Means approximately 850 MB of physical RAM is currently used.

---

# ⚡ CPU Information

Example:

```text
%CPU

42.5
```

Indicates the percentage of CPU time currently being consumed.

Useful for identifying CPU-intensive processes.

# 🐳 Docker Example

Container:

```bash

```


You can also inspect processes inside the container:

```bash
docker exec -it <container-id> ps -ef
```

# 🚫 Common Mistakes

## ❌ Thinking `ps` Shows Historical Processes

`ps` only shows processes that currently exist.

Once a process exits, it disappears from the process table.

---

## ❌ Confusing VSZ and RSS

* **VSZ** = Total virtual address space reserved by the process.
* **RSS** = Portion of that memory currently resident in physical RAM.

RSS is usually the more useful value when investigating memory usage.

---

## ❌ Assuming Every Thread Is a Separate Process

By default, `ps` shows processes.

To display threads:

```bash
ps -eLf
```

Threads share the same process resources but have their own thread IDs.

---

# 📊 Common `ps` Commands

| Command           | Purpose                                             |
| ----------------- | --------------------------------------------------- |
| `ps`              | Show processes attached to the current terminal     |
| `ps -e`           | Show all processes                                  |
| `ps -ef`          | Full process listing                                |
| `ps aux`          | BSD-style process listing with CPU and memory usage |
| `ps -ef --forest` | Display the process hierarchy                       |
| `ps -eLf`         | Display processes and their threads                 |

---

# 💡 Key Takeaways

✅ Every running application in Linux is represented as a process.

✅ The `ps` command displays information about the current process table.

✅ Every process has a unique PID and a parent process (PPID).

✅ `ps -ef` is commonly used to inspect process hierarchies, while `ps aux` emphasizes CPU and memory usage.

✅ Docker containers and Kubernetes Pods ultimately run ordinary Linux processes that can be inspected with `ps`.

✅ Understanding `ps` is the foundation for troubleshooting applications, containers, and Kubernetes nodes.
